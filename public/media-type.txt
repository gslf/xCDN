Type name: application
Subtype name: xcdn
Required parameters: N/A
Optional parameters: charset (default: UTF-8)
Encoding considerations: binary
  XCDN is a textual format but supports native binary literals (b"...").
  It is safe for 8-bit clean transports.
Security considerations: 
  Parsers must enforce limits on nesting depth and numeric precision 
  to prevent resource exhaustion attacks. 
  See Section 6 of the XCDN Specification.
Interoperability considerations: 
  Designed as a superset of JSON's cognitive model. 
  Parsers SHOULD support streaming (concatenated values).
Published specification: https://github.com/gslf/xcdn
Applications that use this media type: 
  Configuration management, log streaming, IoT data exchange, 
  financial API responses.
Fragment identifier considerations: N/A
Additional information:
  Deprecates: application/json (in high-precision contexts)
  Magic number(s): N/A
  File extension(s): .xcdn
  Macintosh file type code(s): TEXT
